Структура файлів проекту
========================

	/fancybox/ - розширення jQuery для виводу і перегляду зображень
		! не змінювати. Джерело: http://fancyapps.com/fancybox/
	/files/ - для зберігання файлів, завантажених на сервер користувачами
	/files/counter - зберігається одне число - останній доданий номер перед файлом
		для забезпечення унікальності імен файлів перед кожним з них буде додаватися
		наступний номер з цього файлу. Напр. якщо число у цьому файлі 1 то наступний 
		файл який буде завантажений буде мати ім'я 2_FileName.ext
	/files/.htaccess - конфігурація сервера для примусового завантаження файлів у цій
		папці, наприклад для того, щоб не відкривати файл pdf у браузері, а відразу 
		його завантажити. Містить список розширень, які потрібно примусово завантажувати.
	/images/ - зображення для оформлення сторінок
	/images/avatars/ - іконки для користувачів, зберігаються у файлі login.jpg де login -
		ім'я користувача, чия це іконка.
	/images/avatars/noAv.png - аватар за замовчуванням, якщо не існує іншого
	/images/icons/ - іконки файлів, використовуються як візуальна інтерпритація файлів
		відображаються для користувачів
	/images/source/ - папка з вихідними файлами зображень, до зменшення, перефарбовування 
		та інших модифікацій, також містить PSD файли для редагування у Photoshop
	/images/SimpleImage.php - php-бібліотека для роботи із зображеннями (зміна розмірів,
		конвертація форматів). !!!Можливо потребує переміщення до includes, через те, що
		не є зображенням. Не повинне модифікуватися, тому краще залишити тут.
	/includes/ - містить файли з функціями потрібними для обробки та виводу даних
	/includes/attaches.php - функції необхідні для прикріплення різних типів об'єктів до
		повідомленнь на головній сторінці
	/includes/files.php - функції необхідні для виведення інформації про файли
	/includes/messages.php - функції необхідні для виведення інформації про повідомлення
	/includes/polls.php - функції необхідні для виведення інформації про опитування
	/includes/userNames.php - отримання повних імен користувачів
	/photos/ - фотографії, завантажені користувачами у фотоальбоми, використовуються
		випадкові назви з 10 цифр та літер, формат jpg
	/photos/thumb/ - зменшені копії зображень з фотогалереї шириною 200px
	/data.php - масив з пунктами меню та деякими стандартними константами, включається в 
		кожну сторінку
	/dbDump.sql - поточний знімок бази даних для імпорту
	/dbModel.png - зображення-модель бази даних лише для кращого розуміння структури
	/favicon.ico - стандартна іконка сайту
	/fileUploader.php - вікно для завантаження файлів
	/files.php - сторінка відображення списку завантажених файлів
	/fonts.css - підключення шрифтів
	/index.php - головна сторінка сайту - список повідомленнь
	/lastMsgTime.php - містить скрипт для виведення одного числа - часу додавання останнього
		повідомлення - для оновлення в потрібний момент
	/login.php - сторінка авторизації користувачів
	/new.ico - іконка, яка відображається при наявності нових повідомлень
	/photos.php - сторінка відображення фотоальбомів
	/polls.php - сторінка відображення опитувань
	/schedule.php - сторінка відображення розкладу
	/script.js - скрипти, клієнська активність
	/style.css - стилі
	/template.php - файл з функціями шаблонів, підключенням до бази даних, отримання інформації
		про поточного користувача
	/uploader.php - вікно завантаження зображень
	/users.php - сторінка відображення списку користувачів
	
Встановлення
============

	Для роботи потрібен веб сервер Apache, інтерпритатор php і база даних MySql
	Можна встановити вже зібрані пакети типу WAMP або Denwer.
	Потрібно імпортувати базу даних із файлу dbDump.sql а також створити користувача
	на сервері MySql з доступом до цієї дази даних. Всі дані необхідно вказати у файлі
	template.php, якщо вони відрізняються від вже вказаних (підключення до бази даних). 

Шаблон сайту
============

	Шаблон сайту збережений у файлі template.php
	він складається з двох функцій printTop($title) і printBottom()
	вони виводять відповідно верхню(меню і бічна панель) і нижню частину сайту
	кожна сторінка повинна включати цей файл (він зокрема містить підключення до бази даних)
	і викликати ці дві функції, між викликами яких повинна виводитись основна інформація
	параметр $title - просто загаоловок сторінки, він визначає, який пункт меню буде вибраний,
	і що буде відображатися у заголовку браузера.
